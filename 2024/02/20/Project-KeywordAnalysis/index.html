키워드, 시간대별 응답률 분석

Created2024-02-20|Updated2024-02-20|Project
|Post Views:

Summary

  • Goal : 데이터 분석 정보 공유방에서 키워드, 시간대별로 응답률을 분석하여 정보를 더욱 효율적으로 얻는 전략 수립
  • 수행기간 : 2023. 11. 21 ~ 2023. 12.27
  • 수행내용
    • 오픈채팅방에서 평균적으로 월요일에 유입/이탈이 활발한 것을 확인하였으며, 주말(토, 일)요일의 경우 유입/이탈 인원이 확연히 감소함
    • 요일별 전체 활동량을 보았을 때 주중(월, 화, 수, 목, 금)의 경우 활동이 활발하게 일어났으며, 주말(토, 일)의 경우 활동량이 미비함.
      • 채팅 참여의 경우 참여자는 317명(65%), 미참여자의 경우 167명(35%)으로 나타남
      • 채팅 참여의 경우 주중에서도 화요일, 수요일에 특히 활동이 많은 것을 확인
    • 채팅 참여자의 직군을 확인하였을 때, 대학생, 데이터분석가, 마케터 직군 순으로 많은 참여자가 분포
      • 경력별로 확인해보았을 때, 경력이 높아질수록 대화자의 수가 적어졌으며, 전체 대화의 33%가 0년차(신입)에서 이뤄지며, 인원대비 대화 참여율은 10년차가 가장 높게 나타남
    • 이탈자 중 채팅 참여 여부를 보았을 때 이탈자 중 채팅 참여비율은 전체 127명 중 14명(11%)로 확인하였으며, 가장 많이 채팅한 이탈자의 경우 7건으로 나타남
    • 질문/답변을 구분하기 위해 적용시킨 ML Model 3가지를 비교하였을 때, 3개의 선정 모델 중 RF(Random Forest)모델이 가장 높은 성능으로 나타남
    • Model Performance를 Accuracy로 봤을 경우는 Random Forest(RF)가 가장 높게 보여주나, 시간비용을 고려한다면 Logistic Regrssion(LR)를 고려하는 것도 좋은 선택지라 생각됨

개요

Background

  • 사용자가 원하는 정보를 얻을 수 있는 방법 중 하나로, 다양한 오픈채팅방이 존재
  • 오픈채팅방을 통하여 정보를 얻거나 공유할 수 있으나, 항상 원하는 답변과 내용을 얻을 수 있는 것은 아니다
  • 질문 및 답변이 활발한 시간대와 원하는 정보와 관련된 키워드를 효율적으로 배치하여 답변을 얻는 전략이 필요

Dataset

  • [데이터리안] 데이터 분석 정보 공유방 (2023-08-13 ~ 2023-12-03)
    • 6,000 rows+

PIPELINE

  • 오픈채팅방 비정형 데이터의 정형화
    • 텍스트 데이터를 정제해서 닉네임/직군/경력/카톡내용/시간으로 구분하여 저장
  • 질문/답변/질문끝 라벨링
  • 카톡 내용 한국어 전처리를 통한 키워드 분류
  • (필요시 Classification) 질문/답변 학습 후 모델 생성
  • 분석 및 인사이트 도출
    • 키워드별 응답률, 활동시간대별 응답률
    • 경력/직군별 키워드 및 응답률




Preprocessing

Raw Data

  • 텍스트 파일(.txt)로 구성
  • 시간, 닉네임, 직무분야, 경력, 대화내용, 입/퇴장 여부 등 다양한 데이터가 존재


데이터 정형화

  • 위의 Raw Data를 활용하여 데이터를 Tabular 형태로 변환
  • Variable


Keyword 분석을 위한 Hand-Labeling

  • 질문과 답변을 구분하기 위해 Hand-Labeling 진행

  • group_num

    • 특정 질문에 대한 대화를 그룹핑
    • 다른 주제에 대해 질의응답을 한 경우 다른 group_num 부어

  • msg_lab

    • 들어온 인원의 경우 0, 나간 인원의 경우 1, 메시지 가리기인 경우 2로 분류 진행

Application

Application with ML

  • 질문을 구분하는 것을 목표로 선정하였으며, 총 3가지의 Maching Learning Model을 통하여 각 모델의 성능을 비교하고자 한다.


  • 전처리된 채팅 데이터를 활용하여 질문 분류 예측 모델을 만듬
  • 이용자의 채팅 내용을 통해 이용자의 채팅이 질문인지 아닌지를 예측하여 이용자의 의도를 파악하는 것
  • 모델의 X 값(즉, feature 값)은 이용자 채팅 내용이 되겠고, 모델의 Y 값(즉, label 값)은 질문 여부
    • 질문과 답변을 라벨링한 것에 대해, 질문일 경우 1, 질문이 아닐 경우 0으로 라벨링을 진행

    • 모델 평가결과를 살펴보면, 모델이 지나치게 질문이 아닌 것(0)으로만 예측하는 경향을 보임
    • 따라서 질문이 아닌 것을 잘 예측하지만, 질문인 것에 대한 예측 정확도가 매우 낮게 나타남
    • 이는 샘플데이터의 클래스 불균형으로 인한 문제로 보임
    • 따라서, 클래스 불균형 조정을 진행


샘플링 재조정

  • 1:1 샘플링

질문/답변 키워드 분석

  • 학습된 Logistic Regression 모델을 이용하여 키워드 추출
    • 추출된 키워드를 통해서 사용자의 의도가 질문인지 아닌지 판단할 수 있고, 채팅의 주 관심사에 대해서도 어느정도 판단할 수 있다.

  • 질문, 질문이 아닌 키워드를 추출하기 위해 먼저 Logistic Regression 모델에 각 단어의 coeficient를 확인

  • 시각화

    • 계수가 양인 경우는 단어가 질문인 경우라고 볼 수 있고, 반면에 음인 경우는 질문이 아닌 키워드라고 볼 수 있다.
    • 이 계수들을 크기순으로 정렬하면, Quesiton / Nonquestion 키워드를 출력하는 지표로 사용
  • 전체 단어가 포함한 ‘질문 키워드 리스트’와 ‘질문이 아닌 키워드 리스트’를 정의하고 출력
  • index를 단어로 변환하여 ‘질문 키워드 리스트’와 ‘질문이 아닌 키워드 리스트’의 Top 20 단어를 출력
  • 키워드를 살펴보면
    보통 혹시, 질문, 관련, 경험, 경우, 해결 등 직무와 관련된 분야에 대한 해결 경험 등을 질문하는 것으로 보임
    하지만 답변, 아하, 한번, 말씀, 감사, 생각 등 질문이 아닌 키워드에서는 주로 질문에 대해 답변하는 키워드로 보임

Conclusion

시간에 따른 유입/활동량

  • 오픈채팅방에서 평균적으로 월요일에 유입/이탈이 활발한 것을 확인하였으며, 주말(토, 일)요일의 경우 유입/이탈 인원이 확연히 감소함
  • 요일별 전체 활동량을 보았을 때 주중(월, 화, 수, 목, 금)의 경우 활동이 활발하게 일어났으며, 주말(토, 일)의 경우 활동량이 미비함
    • 전체적으로 주중에 공통적으로 활발하게 활동하는 것으로 나타났으나, 특정 요일에 활발하다는 결론은 내지 못함


채팅 참여

  • 활동시간(09시 ~ 18시) 내 커뮤니케이션이 활발하게 일어나며, 특히, 15 ~ 17시 가장 높은 분포를 보임

    • 채팅 참여의 경우 참여자는 317명(65%), 미참여자의 경우 167명(35%)으로 나타남
    • 채팅 참여의 경우 주중에서도 화요일, 수요일에 특히 활동이 많은 것을 확인

  • 채팅 참여자의 직군을 확인하였을 때, 대학생, 데이터분석가, 마케터 직군 순으로 많은 참여자가 분포

    • 경력별로 확인해보았을 때, 경력이 높아질수록 대화자의 수가 적어졌으며, 전체 대화의 33%가 0년차(신입)에서 이뤄지며, 인원대비 대화 참여율은 10년차가 가장 높게 나타남

  • 이탈자 중 채팅 참여 여부를 보았을 때 이탈자 중 채팅 참여비율은 전체 127명 중 14명(11%)로 확인하였으며, 가장 많이 채팅한 이탈자의 경우 7건으로 나타남

  • 채팅 키워드로 ‘데이터’, ‘이벤트’, ‘프로젝트‘, ’페이지‘ 단어의 경우 전체 채팅과 질문자의 채팅에서 두드러지게 나타남
  • 답변 수를 시간대별로 비교하였을 때, 10:00, 15:00, 09:00 순으로 가장 많은 답변을 보임


질문/답변 구분

  • 질문/답변을 구분하기 위해 적용시킨 ML Model 3가지를 비교하였을 때, 3개의 선정 모델 중 RF(Random Forest)모델이 가장 높은 성능으로 나타남
  • 질문/답변을 구분하는 키워드를 보았을 때, 혹시, 질문, 관련, 경험, 경우, 해결 등 직무와 관련된 분야에 대한 해결 경험 등을 질문하는 것으로 보임. 답변, 아하, 한번, 말씀, 감사, 생각 등 질문이 아닌 키워드에서는 주로 질문에 대해 답변하는 키워드로 보임
  • Model Performance를 Accuracy로 봤을 경우는 Random Forest(RF)가 가장 높게 보여주나, 시간비용을 고려한다면 Logistic Regrssion(LR)를 고려하는 것도 좋은 선택지라 생각됨

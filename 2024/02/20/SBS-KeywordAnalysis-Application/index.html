(Step) 키워드, 시간대별 응답률 분석 - Application

|Post Views:

Application with ML

  • 질문을 구분하는 것을 목표로 선정하였으며, 총 3가지의 Maching Learning Model을 통하여 각 모델의 성능을 비교하고자 한다.


  • 전처리된 채팅 데이터를 활용하여 질문 분류 예측 모델을 만듬
  • 이용자의 채팅 내용을 통해 이용자의 채팅이 질문인지 아닌지를 예측하여 이용자의 의도를 파악하는 것
  • 모델의 X 값(즉, feature 값)은 이용자 채팅 내용이 되겠고, 모델의 Y 값(즉, label 값)은 질문 여부
    • 질문과 답변을 라벨링한 것에 대해, 질문일 경우 1, 질문이 아닐 경우 0으로 라벨링을 진행


  • 라벨링(qna_cnt) 된 행의 갯수 파악
  • Train, Test Set 분리


Logistic Regression Model

  • Logistic Regression Model 학습
  • 분류 결과 평가
  • 모델 학습 결과
Skill Score Value
Accuracy 0.86
Precision 1.00
Recall 0.11
F1 score 0.20
  • Confusion Matrix
    • 모델 평가결과를 살펴보면, 모델이 지나치게 질문이 아닌 것(0)으로만 예측하는 경향을 보임
    • 따라서 질문이 아닌 것을 잘 예측하지만, 질문인 것에 대한 예측 정확도가 매우 낮게 나타남
    • 이는 샘플데이터의 클래스 불균형으로 인한 문제로 보임
    • 따라서, 클래스 불균형 조정을 진행


샘플링 재조정

  • 1:1 샘플링
    • 과적합(Overfitting)을 방지하기 위해 샘플의 갯수를 1:1 비율로 수정


LR Model 재학습

  • 모델 평가
  • 모델 학습 결과
Skill Score Value
Accuracy 0.84
Precision 0.84
Recall 0.80
F1 score 0.82
  • Confusion Matrix
    • 모델이 ‘질문’ 케이스와 ‘질문이 아닌’ 케이스를 모두 적당히 잘 맞춘 것을 확인


질문/답변 키워드 분석

  • 학습된 Logistic Regression 모델을 이용하여 키워드 추출
    • 추출된 키워드를 통해서 사용자의 의도가 질문인지 아닌지 판단할 수 있고, 채팅의 주 관심사에 대해서도 어느정도 판단할 수 있다.
  • 질문, 질문이 아닌 키워드를 추출하기 위해 먼저 Logistic Regression 모델에 각 단어의 coeficient를 확인
  • 시각화
    • 계수가 양인 경우는 단어가 질문인 경우라고 볼 수 있고, 반면에 음인 경우는 질문이 아닌 키워드라고 볼 수 있다.
    • 이 계수들을 크기순으로 정렬하면, Quesiton / Nonquestion 키워드를 출력하는 지표로 사용
  • 전체 단어가 포함한 ‘질문 키워드 리스트’와 ‘질문이 아닌 키워드 리스트’를 정의하고 출력
  • index를 단어로 변환하여 ‘질문 키워드 리스트’와 ‘질문이 아닌 키워드 리스트’의 Top 20 단어를 출력
  • 키워드를 살펴보면
    보통 혹시, 질문, 관련, 경험, 경우, 해결 등 직무와 관련된 분야에 대한 해결 경험 등을 질문하는 것으로 보임
    하지만 답변, 아하, 한번, 말씀, 감사, 생각 등 질문이 아닌 키워드에서는 주로 질문에 대해 답변하는 키워드로 보

데이터 구성

지역 설정

  • 아래 시간 및 지역에 해당하는 Sea Surface Temperature(SST) Data Download
  • 시간 : 2023. 01. 01. ~ 2024. 01. 01
  • 경도 : 124° ~ 132°, 위도 : 33° ~ 43°


  • 파일


시간(Time)

  • 시간 구성을 확인하기 위해 head 확인
    • 지수표기법으로 구성
  • (필요시) 지수표기법을 일반화하여 확인
    • OSTIA Time의 경우 에포크 시간(epoch time)으로 표현되는 것을 확인

에포크 시간(epoch time)

  • 에포크 시간은 보통 1970년 1월 1일 0시 0분 0초 (UTC)부터 현재까지의 시간을 초 단위로 표현한 것
  • 따라서 이 숫자들은 해당 기간동안의 초 단위를 나타냄
  • 에포크 시간을 현재 시간으로 변환
    • time 변수의 경우 masking numpy array로 되어있어 data만 추출하여야 변환 가능
  • 변환된 시간을 date 형태로 변환
    • 변환된 시간을 'yyyy-mm-dd'와 'hh:mm'으로 변환


공간(Space)

  • 공간을 나타내는 변수인 위도(latitude)와 경도(longitude)를 확인
  • 우리가 보고자 하는 변수 ‘analysed_sst‘에 대한 데이터 정보 확인
    • 3-dimension으로 구성되어있으며, (time, latitude, longitute)를 나타냄
    • time의 경우, 일(day) 단위를 나타냄


Visualization

  • 위/경도에 따른 수온을 시각화

수온의 상태가 이상하다

  • OSTIA NetCDF 수온의 경우, 켈빈(Kelvin, K)으로 단위가 지정되어 있음
  • 따라서, 켈빈을 섭씨(Celcius, $ ^oC$)로 변경해줄 필요가 있음
  • 수온 단위 변환 및 시각화
  • colorbar 고정을 위해 전체 데이터의 Max, Min(즉, 최대, 최소 표층수온) 구하기
  • 섭씨(Cellcius)로 변환된 값을 이용해 Figure Out

NetCDF to Tabular

NetCDF로 구성되어있는 파일을 Tabular Data로 변환하는 작업 수행

  • 차후 데이터 확인을 위해 Tabular가 시각적으로 더 좋을 것이라 판단
  • 딥러닝 모델을 적용시키기 위해서도 Tabular가 더 용이할거라고 생각함
  • 표층수온(Sea Surface Temperature, SST)의 Data shape 검토
    • (time, lat, lon)으로 구성
    • 즉, 일별 전체 표층수온 데이터가 한 channel에 존재
    • ex) temp_K[0, :, :] : 첫째 날(2023. 01. 01) 표층수온 전체
  • 또한, 일별 데이터에 해당하는 시간을 나타내는 변수는 time에 할당되어있음

정리

  • 데이터를 매칭함에 있어, time.data의 첫째 항(시간)에 해당하는 표층수온의 데이터는 temp_K[0, :, :]가 되며, 그에 해당하는 latitude와 longitude의 정보는 각각 lats, lons에 저장되어있음

ex) 특정 날 특정 latitude, longitute의 수온을 보고싶다.

- 특정 시간 : `time.data[days]`
- 특정 latitude의 위치 : `lats[ilat]`
- 특정 longitude의 위치 : `lons[ilon]`
- 표층수온(SST) : `temp_K[days, ilat, ilon]`

주의

  • NetCDF를 Tabular로 변환은 성공하였으나, 실제로 데이터 매치가 잘 되었는지 확인해야 함
    • 코드는 잘 작동하였으나, 시공간적 데이터 정보가 다르게 들어가면 안되기 때문
  • 따라서 위에서 시각화한 내용을 Tabular로도 똑같이 구현이 되는지 확인 필요!
  • tmp DataFrame을 만들어 Test 진행
    • 데이터 기준 첫 날(2023-01-01) 데이터를 시각화 진행
    • 하루 데이터 : lon $\times$ lat = 32,000개
  • colormap을 그리기 위해 데이터를 2D Grid로 변환
  • 마스킹 데이터 처리 진행
    • 위 Tabular를 살펴보면, ‘--‘로 되어있는 구간이 존재
    • 해당 Value는 문자, 결측값이 아닌 Masked Constant라는 변수로 존재(Dtype : Object)
    • 따라서, nan값으로 변환 후, 숫자형으로 바꿔주는 작업이 필요
    • 시각화를 위해 마스킹 처리가 된 numpy masked 배열로 변환
  • 시각화 진행
    • 기존 NetCDF 파일과 Tabular Data를 각각 시각화한 결과 일치하는 것을 보임(Max SST, Min SST 확인)

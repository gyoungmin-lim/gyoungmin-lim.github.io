LSTM 기법을 활용한 수온 예측

Created2024-02-22|Updated2024-03-26|Project
|Post Views:

Summary

  • Goal : LSTM 기법을 활용한 수온 예측
  • 수행기간 : 2024.01.31 ~ 2024.03.21
  • 수행내용
    • 위성데이터의 경우 Epoch Time으로 이루어져 있어 Local Time으로 변환
    • 시간에 따른 해수면온도(Sea Surface Temperature, SST) 변화를 살펴본 결과, 여름철 남쪽에서부터 북쪽으로 SST가 상승하며, 반대로 겨울철에 북쪽에서 남쪽으로 SST 하락을 보임. 이는 해류와 연관성이 있어보임.
    • 사용모델 : LSTM(Long Short Term Memory)
    • 평가지표 : RMSE(Root Mean Square Error)
    • 예측 결과, 연안에서 RMSE가 대체적으로 높게 나타나는 것을 확인하였으며, 원해의 경우 남해 부근에서 RMSE가 높게 나타남
    • 장기 데이터를 사용할수록 RMSE가 증가하는 것을 확인. 따라서 단기 예측에는 효율적이나, 장기 예측의 경우 부정확할 것이라고 판단됨. 그러나 중간에 예측값(Day4, Day6)이 튀는 현상은 해결하지 못함
    • 추후 Drop-Out Method 등을 활용하여 해당 문제를 해결할 수 있을 것이라고 생각됨


개요

  • Sea Surface Temperature(SST) : 지구 해양-대기 시스템에서 중요한 매개변수 중 하나
  • 지구 바다 전역 기후에 깊은 영향을 미침
  • 강수량 분포에 영향을 미쳐 가뭄과 홍수 같은 심각한 피해를 끼치는 기상사건 발생 가능성이 커짐


OSTIA(Operational Sea Surface Temperature and Sea Ice Analysis)

  • 해양수치모델 OSTIA는 해양의 표면 온도와 해빙 분포를 모니터링하고 예측하기 위한 모델

  • 영국 해양과학 연구소 (UK Met Office)에서 개발하고 운영

  • 위성 자료를 기반으로 해양 환경을 추정

  • 표면 온도와 해빙을 모니터링하고 예측하는 데에 사용
    • 해양 수면 온도와 해빙은 해양 생태계, 기후 변화, 해양 탐사 및 해양 자원 관리 등 다양한 분야에서 중요한 역할
  • 국제 해양 연구 및 모니터링 프로그램에서 널리 사용
  • 해양 환경에 대한 이해를 향상시키고 다양한 관련 분야에서의 응용 가능성을 확대하는 데 기여

특징

특징 내용
위성 자료 활용 위성 자료를 기반으로 해양 환경을 모니터링하고 예측하는 데 사용
넓은 지역의 해양 상태를 실시간 또는 예측
공간 및 시간 분해능 OSTIA는 고해상도의 공간 분해능과 시간 분해능을 제공하여 해양 환경을 상세하게 파악
실시간 및 예측 실시간 및 예측 모드에서 작동하여 해양 환경의 현재 상태를 모니터링하고 미래 상황을 예측
해양 관련 활동을 계획하고 관리하는 데 도움
다양한 응용 분야 해양 예측 및 모니터링, 해양 생물학적 연구, 해양 자원 개발 등 다양한 분야에 활용
국제적 활용 국제 해양 연구 및 모니터링 프로그램에서 널리 사용
다양한 국가 및 기관에서 활발하게 활용
  • 공간 분해능
    • 데이터나 모델이 특정 지역을 얼마나 세세하게 분석할 수 있는지를 나타냄
    • 공간 분해능이 높을수록 더 작은 지리적 영역을 상세하게 파악 가능
    • ex) 고해상도 위성 이미지는 더 작은 지형 특징이나 객체를 분석하는 데 유용합니다.
  • 시간 분해능
    • 데이터나 모델이 시간을 얼마나 세밀하게 측정하거나 예측할 수 있는지를 나타냄
    • 시간 분해능이 높을수록 더 짧은 시간 간격으로 변화를 관찰하거나 예측할 수 있다
    • ex) 기상 예측 모델이 날씨 변화를 미래 몇 시간 내에 얼마나 정확하게 예측할 수 있는지에 영향

▶︎ 고해상도의 공간 분해능과 시간 분해능을 가진 모델은 해양 환경을 상세하게 파악할 수 있고, 실시간 모니터링과 예측에 유용

Data Configuration

지역 설정

  • 아래 시간 및 지역에 해당하는 Sea Surface Temperature(SST) Data Download
    • 시간 : 2023. 01. 01. ~ 2024. 01. 01
    • 경도 : 124° ~ 132°, 위도 : 33° ~ 43°


시간(Time)

  • 시간 구성을 확인하기 위해 head 확인
    • 지수표기법으로 구성되어있다. 따라서 지수표기법을 일반화하여 확인
    • OSTIA Time의 경우 에포크 시간(epoch time)으로 표현되는 것을 확인

에포크 시간(epoch time)

  • 에포크 시간은 보통 1970년 1월 1일 0시 0분 0초 (UTC)부터 현재까지의 시간을 초 단위로 표현한 것
  • 따라서 이 숫자들은 해당 기간동안의 초 단위를 나타냄
  • 에포크 시간(epoch time)을 현재 시간(local time)으로 변환
    • time 변수의 경우 masking numpy array로 되어있어 data만 추출하여야 변환 가능
    • 변환된 시간을 date 형태로 변환
    • 변환된 시간을 'yyyy-mm-dd'와 'hh:mm'으로 변환


공간(Space)

  • 공간을 나타내는 변수인 위도(latitude)와 경도(longitude)를 확인
위도(latitude) 경도(longitude)


Sea Surface Temperature(SST)

  • 우리가 보고자 하는 변수 ‘analysed_sst‘에 대한 데이터 정보 확인
    • 3-dimension으로 구성되어있으며, (time, latitude, longitute)를 나타냄
    • time의 경우, 일(day) 단위를 나타냄




EDA

단위 변환

  • 위/경도에 따른 표층수온(SST)을 시각화

수온의 상태가 이상하다

  • OSTIA NetCDF 수온의 경우, 켈빈(Kelvin, K)으로 단위가 지정되어 있음
  • 따라서, 켈빈을 섭씨(Celcius, $ ^oC$)로 변경해줄 필요가 있음
  • 섭씨(Cellcius)로 변환된 값을 이용해 Figure Out

계절성 변화도

  • 수온의 변화를 눈에 잘 보이게 하기 위해, 표층수온 전체 데이터의 Max, Min(즉, 최대, 최소 표층수온) 구한 후, colorbar 고정

    • 가지고 있는 데이터의 최대, 최소값으로 설정
    • Max : 31.25, Min : -2.0

  • 계절 기준 표층수온(SST) 변화도를 확인하기 위해 봄(3, 4, 5월), 여름(6, 7, 8월), 가을(9, 10, 11월), 겨울(12, 1, 2월)의 첫 날짜를 기준으로 시각화를 진행

Application with LSTM

  • LSTM 모델을 활용하여 과거 데이터를 얼만큼 사용했을 때의 결과를 비교하기 위해 코드를 진행하기 앞서, 한 가지를 먼저 진행
  • Sequence Length = 3, 즉 3일간의 데이터를 사용했을 때의 결과를 먼저 비교하고자 함
  • 이후 Sequence Length의 변화에 따른 결과를 비교

Lead 3 time

  • LSTM을 구현하기 위해, 현재 데이터프레임에 생성되어있는 NaN column 제거
    • 모델을 돌리기 위한 데이터프레임에 NaN값이 포함되어있을 경우 동작하지 않는 것으로 알고 있다.
    • 현재 데이터프레임은 masking되어 있는 값으로 구분되기 때문에, masking patern을 가지고 있는 경우 제거하기 위해 column list를 작성하여 추가해준다.


LSTM Model 구축

  • Sequence Length 설정 및 Train/Test Set 분리
  • 확인결과, 2023.10.20.을 기준으로 각각 Set 분리되는 것을 확인
    • Sequence Length : 3
    • Train : Test = 8 : 2
      • 2023.10.20 이전 : Train Set
      • 2023.10.20 이후 : Test Set


  • 구축된 모델을 기반으로 예측 값과 실제 값 시각화하여 비교
    • 여기서 실제 값(y)은 OSTIA 관측 데이터를 의미
실제값(y) 예측값(prediction)


공간별 예측결과

  • 예측된 결과를 비교하기 위해, 예측값과 실제값의 차이를 시각화
    • 2023년 10월 20일 기준 결과 비교

RMSE(Root Mean Square Error)

  • 모델 성능을 평가하기 위해, 공간별 RMSE 시각화

    • 각 좌표(lon, lat)별 RMSE를 시각화

  • 구축한 모델을 기반으로 예측을 진행하며, 모델의 성능을 평가하기 위해 RMSE(Root Mean Square Error)를 평가지표로 활용

평균 제곱근 오차(Root Mean Square Error; RMSE)

  • 추정 값 또는 모델이 예측한 값과 실제 환경에서 관찰되는 값의 차이를 다룰 때 사용하는 측도
  • 관측 값과 예측 값 사이 오차의 절댓값을 활용한 지표where $n$ is # of data, $y_i$ is reference of OSTIA, $x_i$ is prediction value
  • 시각화 결과
    • 기간 : 2023.10.20 ~ 2024.01.01
    • 대체적으로 연안에서 RMSE가 큰 것을 확인
    • 원해의 경우, 한반도 남해에서 RMSE가 큰 것을 확인

Conclusion

데이터 구성

  • 위성데이터의 경우 Epoch Time으로 이루어져 있어 Local Time으로 변환
  • 또한 딥러닝 모델을 적용시키기 위해 NetCDF 파일을 Tabular 형태로 변경

EDA

  • 시간에 따른 해수면온도(Sea Surface Temperature, SST) 변화를 살펴본 결과, 여름철 남쪽에서부터 북쪽으로 SST가 상승하며, 반대로 겨울철에 북쪽에서 남쪽으로 SST 하락을 보임. 이는 해류와 연관성이 있어보임.

Application with LSTM

  • 예측 결과, 연안에서 RMSE가 대체적으로 높게 나타나는 것을 확인하였으며, 원해의 경우 남해 부근에서 RMSE가 높게 나타남
  • 장기 데이터를 사용할수록 RMSE가 증가하는 것을 확인. 따라서 단기 예측에는 효율적이나, 장기 예측의 경우 부정확할 것이라고 판단됨. 그러나 중간에 예측값(Day4, Day6)이 튀는 현상은 해결하지 못함
    • 추후 Drop-Out Method 등을 활용하여 해당 문제를 해결할 수 있을 것이라고 생각됨
  • 시간적 데이터량(현재 데이터셋은 1년)이 적어 계절별 특징을 확인하지 못함
    • 현재는 2023년 10월 20일 이후 데이터셋으로만 비교 가능
    • 추후 데이터셋을 확장한다면 계절별 예측 결과를 확인할 수 있을 것으로 보임

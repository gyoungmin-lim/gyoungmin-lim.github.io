(Git) Remote Repository

Created2024-02-18|Updated2024-02-18|ToolGit
|Post Views:

Remote Repository 생성

  • Create repository
  • 프로젝트 이름 설정 뒤 빈 프로젝트 상태로 Create repository 클릭
  • Remote Repository 생성 확인


Github Token 생성

보안상의 이유로 Remote Repository 접속 시 비밀번호 대신 Token 사용

  • 사용자 메뉴에서 Setting 선택
  • 좌측 하단 Developer Setting 선택
  • Personal access tokens 클릭
  • 아래와 같이 설정
  • 생성 완료

(참고) Read Me

  • 프로젝트에 대한 설명, 사용방법, 라이센스, 설치방법 등에 대한 내용을 기술하는 파일
  • 나, 직장동료, 프로그램 사용자를 위해 존재


(참고) .gitignore

  • Git 버전 관리에서 제외할 파일목록을 지정하는 파일
  • 사용자가 원하지 않는 파일들을 자동으로 commit 대상에서 제외시켜 줌


Remote Repository 등록

  • Local Repository에 Remote Repository 등록
  • ID와 PW를 항상 입력해야하는 문제 발생. 이를 해결하기 위해 아래와 같은 문법 사용


Default Repository

  • Main or Master
  • 수정이 가능
    • 수정은 신중해야 한다. 다른 팀원들까지 영향을 받는다.


View All Branches


Default Branch 설정

  • Remote Repository 를 생성할 때 Default Branch 이름을 설정할 수 있음
  • 사용자 메뉴 > Settings


Remote Repository 복제

  • Local Repository 를 생성하지 않은 상태에서 Git Clone 명령을 사용하여 Remote Repository 를 Local 에 복제할 수 있음


Git Clone

  1. 폴더 생성
  2. Git Init 으로 해당 폴더를 초기화
  3. Remote Repository 를 등록
  4. Remote Repository 의 내용을 Pull 하는 모든 과정을 Git Clone 으로 할 수 있음
  • Git Clone with username and token


Application

  • 연결하기


Remote Repository에 변경내용 Push하기

  • Local Repository(HEAD)에 반영된 변경내용을 Remote Repository에 반영하기 위해 git push사용
  • Remote Repository 확인
    • Remote Repository 페이지에서 새로고침 하면 Push된 파일이 보임


Remote Repository에서 Pull 하기

  • Local Repository(HEAD)에 반영된 변경내용을 Remote Repository에 반영하기 위해 git pull 사용
  • ls를 통해 확인
    • 파일이 있을 경우, 잘 된 것


Git Clone 실습

  • Remote Repository 주소 복사
    • 코드를 클릭하면 주소가 나옴
  • 수행 후 확인

야구선수 연봉 예측

Created2024-02-21|Updated2024-02-21|EDA
|Post Views:

Summary

  • Goal : 야구선수 ‘최동원’의 연봉 예측하기
  • 수행기간 : 2023. 12. 11 ~ 2023. 12. 18
  • 수행내용
    • 결과
      • 연봉과 상관관계가 0.2 이상인 Feature들을 추출한 LinearRegression(cor>0.2)와 RidgeRegression(cor>0.2)를 보았을 때, 최동원 선수의 1983-1988년 선수기록으로 보아 각 연도 연봉은 각각 1983년 : 8.25억, 1984년 : 13.1억, 1985년 : 10.4억, 1986년 : 12.5억, 1987년 : 9.8억으로 예상된다.
      • 모델 성능이 전체적으로 좋지 않으나… r2_score < 0 인 DT Regression을 제외하며, Linear Regression 모델에서 연봉 또한 음수가 나오게 되므로 제외
    • 문제점 및 해결방안
      • 물가상승률을 적용하였을 때, 1980년대 당시 연봉으로 변환할 수 있을 것 같음
      • 모델 성능을 올려보기 위해, scaler를 시도했으나 r2_score를 기준으로 별다른 변화가 없었다.
      • 적은 데이터 및 데이터 산포도가 넓어 발생한다고 생각한다.


데이터 수집

  • Module Load


연봉 데이터

  • STATIZ
    • 링크 : http://www.statiz.co.kr/salary.php?opt=0&sopt=2020&te=
  • 웹 페이지의 형태가 연도별 링크로 되어있다.

선수 경기데이터

  • Korea Baseball Organization (KBO)

    • 링크 : https://www.koreabaseball.com/Record/Player/PitcherBasic/Basic1.aspx?sort=ERA_RT

  • 선수 경기기록을 구할 수 있다.

    • 연도별 선수 경기기록 데이터를 가져오기

  • column명 확인

    • 1985~1988년과 2015~2020년 선수기록 column명이 다른 것을 확인
    • 추후 데이터 병합 후 전처리 시 참고

데이터 저장

  • 연봉 데이터와 경기기록 데이터를 저장


데이터 전처리

데이터 병합

  • 1983년 ~ 1988년 데이터와 2015년~2020년 데이터의 칼럼이 다르므로 최신 데이터의 칼럼을 기준으로 데이터 통합

  • 데이터 병합을 위해 column명 변경

    • 연봉데이터의 column명을 선수명으로 변경

  • 선수기록 데이터를 기준으로 데이터 병합

    • 연도, 선수명 column을 기준으로 데이터 병합


column 제거

  • 1983년~1988년 데이터와 2015년~2020년 데이터의 컬럼 구성이 다름을 확인

    • 구 데이터(1983년~1988년)
      • 순위 선수명 팀명 ERA G CG SHO W L SV HLD WPCT TBF IP H HR BB HBP SO R ER
    • 최신 데이터(2015년~2020년)
      • 순위 선수명 팀명 ERA G W L SV HLD WPCT IP H HR BB HBP SO R ER WHIP

  • 최신 데이터에서 CG, SHO 기록이 없음

    • 최신 데이터인 2015년~2020년 데이터의 컬럼 구성
    • 데이터 중 IP 컬럼의 경우 숫자 + 분수의 문자형(str)로 되어 있어, 이를 변환
    • IP 컬럼 str ▶︎float형 변환

시각화(Visualization)에 앞서, 과거 데이터(1985~1988)의 경우 NaN값이 많아 모델의 성능에 악영향을 끼칠 것으로 판단하여 최신 데이터(2015~2020)를 사용하기로 함

  • 각 element의 상관관계(correlation)을 히트맵으로 표현
    • 선수의 status를 나타내는 요소로서 아래와 같이 지정
      • ‘연도’, ‘연봉(만원)’, ‘WAR’, ‘순위’, ‘ERA’, ‘G’, ‘W’, ‘L’, ‘SV’, ‘HLD’, ‘WPCT’, ‘H’, ‘HR’, ‘BB’, ‘HBP’, ‘SO’, ‘R’, ‘ER’, ‘WHIP’, ‘Processed_IP’
    • SO, Processed_IP, HBP, W, WAR 순으로 연봉과 높은 상관관계를 보임
  • 예측모델을 돌리기 앞서, column 확인
    • 최동원 선수의 데이터를 확인 결과, WAR, WHIP의 정보가 없음(NaN)
    • 해당 column을 미활용하도록 함


머신러닝 모델을 통한 연봉 예측

  • 최동원 선수
    • main dataframe에서 최동원 선수의 선수정보를 변수에 저장
    • WAR, WHIP column 및 연봉의 경우 NaN값으로 채워져 있어 column 삭제 (불필요하다 판단)
    • 1983~1987년 총 5년의 데이터가 존재함

Model별 소개 전 공통사항

  • Train : Test 비율은 8 : 2로 설정
  • random_state = 19로 고정

예측결과

  • 모델 성능이 전체적으로 좋지 않으나….. r2_score < 인 DT Regression을 제외하며, LinearRegression 모델에서 연봉 또한 음수가 나오게 되므로 제외

문제점 및 해결방안

  • 물가상승률을 적용하였을 때, 1980년대 당시 연봉으로 변환할 수 있을 것 같음

  • 적은 데이터 및 데이터 산포도가 넓어 발생한다고 생각한다.

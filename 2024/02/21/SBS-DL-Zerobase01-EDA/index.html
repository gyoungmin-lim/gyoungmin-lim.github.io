Openpose - Body25

Try : Head + Neck

  • 상반신에서도 머리와 목 부분만 Observation을 확인해보기 위해 그려봄
    • 데이터 확인을 위해 1번 사람의 1그룹 카메라 1번에서 관측되는 값을 사용
    • 각 Figure를 그림에 있어 title에 해당하는 신체부위를 작성, x축은 프레임으로 통일하였으며, y축은 위부터 순서대로 결과 도출의 x축, y축, z축으로 통일


Mediapipe - Brazepose

  • Module Load
  • Figure 저장소 설정
  • Data Load


Target Point 선정

  • 우리가 생각하는 ‘쓰러지기’의 경우, ‘서 있는 상태’에서 ‘누워있는 상태’로 전환되는 것을 생각
    • 따라서, 머리의 움직임이 가장 클 것으로 생각하여 머리에 해당하는 점을 Target point로 지정
  • 시각화를 위한 축 데이터 생성
  • 시각화

중간점검

  • 시각화 결과, 전반적으로 비슷한 트렌드의 그래프를 가짐
  • 따라서, 머리(Head)의 모든 점을 보는 것은 비효율적이라 판단
  • 새로운 Target Point 지정


New Target Point

  • Target Point 재선정
    • 몸 전체 움직임을 확인하기 위해 5 point로 축소 및 재선정
    • 가장 큰 움직임을 보여줄 것으로 예상
Previous Target New Target


Figure Out

같은 카메라 시점으로 보기

  • 카메라 시점에 따른 데이터 변화를 보기 위해, 임의적으로 인덱스별 한 개의 데이터 셋을 선정
    • Index 17번 (머리 드라이기로 말리기) 중 Index 17번 Group 1 Camera 1 데이터 탐색
  • 시간(Frame)에 따른 각 point 상/하 움직임을 그래프로 표현
    • 데이터의 경우 양 팔의 움직임이 발생
  • 시간(Frame)에 따른 각 point 원/근 움직임을 그래프로 표현
    • 난잡한 그래프가 발생
  • 같은 방법으로, Index 53(쓰러지기)의 움직임 탐색
  • Frame당 각 Point 상/하 움직임을 그래프로 표현
    • (좌) 그래프의 경우, 일정한 움직임(거의 변화없음)을 보임
    • (우) 그래프의 경우, 난잡한 움직임을 보임
상/하 움직임 원/근 움직임

중간점검

  • Index 53 쓰러지기 데이터의 경우, 쓰러지는 Motion이 아닌, 쓰러진 후 누워있는 상태의 영상만 존재
  • ‘ 활동 중 ‘인 상태와 ‘ 주의/경고 를 나타내는 상태로 구분하고자 목표를 변경


특정 기준 데이터 탐색

  • 촬영장소 및 카메라 위치별 추출된 좌표의 비교를 위해 특정 기준을 정하여 데이터 비교

    • 기준 : 특정 인덱스 인원 1명
    • 그룹별 추출 데이터 트렌드 비교
      • 여기서 그룹(Group)은 촬영된 위치를 나타낸다. (위 참조)

  • Index 17번 P20 인원의 촬영 영상을 그룹별로 나눠 비교

    • 비교 결과, Group별로 다른 트렌드의 그래프가 나타남
  • 같은 방법으로, Index 53 (쓰러지기)에 대한 데이터 탐색
  • P19 인원의 촬영 영상을 나눠 비교
    • Group, 즉 장소에 상관없이 일정한 경향의 그래프를 나타냄

Extracting Model

OpenPose

  • OpenPose는 신체 동작 및 포즈를 감지하는 오픈 소스 라이브러리
  • CNN(Convolutional Nerual Network) 기반으로 관절과 뼈대를 예측
  • 신체의 2D 포즈를 추정하는 데 사용되며, 여러 개의 관절 위치를 식별하여 사람의 동작을 추적
  • 실시간으로 사람의 관절 위치를 추적하고, 이를 통해 다양한 응용 분야에서 활용
    • 주로 컴퓨터 비전과 인공 지능 분야에서 사용
    • 운동 분석, 모션 캡처, 게임, 의료 분야 등 다양한 분야에서 적용
  • 깊은 신경망(deep neural network)을 사용하여 이미지나 비디오에서 사람의 신체 부위를 감지하고 추적합니다. 이를 통해 머리, 어깨, 손, 발 등 다양한 관절의 위치를 정확하게 파악

Body-25

  • 신체 부위의 예측을 위해 25개의 관절로 이루어진 키포인트를 사용
  • 각 관절은 사람의 신체 부위를 나타내며, 이를 통해 다양한 동작을 인식하고 추적할 수 있습니다.
  • 그림 참조
  • (left) COCO-18 model (right) Body-25 model 추정 key points


Variable Meaning

  • frameNum
    • 현재 프레임의 번호 또는 인덱스
    • 시간의 경과에 따라 증가
  • bodyIndexID
    • 추적 중인 사람 또는 몸체의 고유 식별자
    • 다수의 사람이 인식되었을 때, 각 사람은 고유한 bodyIndexID를 가짐
  • trackingID
    • 추적 중인 사람 또는 몸체에 할당된 추적 식별자
    • 시간이 지남에 따라 변경될 수 있다

  • joint_3dX, joint_3dY, joint_3dZ

    • 3차원 공간에서 첫 번째 관절의 X, Y, Z 좌표, 즉 관절의 3D 공간 좌표를 나타내는 변수
      ex) joint1은 Nose(코) 관절에 해당

    • 일반적으로, 좌표계의 원점이 카메라 또는 센서의 위치에 따라 결정됨. 즉 3D 좌표값은 원점으로부터 어떤 방향으로 얼마나 떨어져 있는지를 나타냄

      • X 축: 왼쪽에서 오른쪽으로 증가 (오른쪽이 양수, 왼쪽이 음수)
      • Y 축: 아래에서 위로 증가 (위가 양수, 아래가 음수)
      • Z 축: 카메라에서 멀어지면 양수, 가까워지면 음수

  • joint_depthX, joint_depthY: 깊이 데이터에서 추출된 관절의 X, Y 좌표

  • joint_orientationX, joint_orientationY, joint_orientationZ, joint_orientationW

    • 관절의 방향을 나타내는 회전 행렬 또는 쿼터니언 값 이 정보는 관절이 어떻게 회전하는지를 나타냄

  • joint_trackingState

    • 관절의 추적 상태를 나타내는 변수로, 관절이 추적되면 일반적으로 1로 표시되며, 추적되지 않으면 0으로 표시
    • 변수가 2의 값을 가질 때, 이는 해당 관절이 예측되었지만 추적되지 않았음을 나타냄
    • 이러한 경우, 모델이 해당 관절을 감지하였지만 실제로 추적할 수 있는 데이터가 없는 상태


Mediapipe

  • 구글에서 주로 인체를 대상으로 하는 비전인식 기능들을 AI모델 개발과 기계학습까지 마친 상태로 제공하는 서비스
  • 몸의 움직임을 33가지 point를 통해 나타내는 Brazepose Model 선정




Extracting Data

  • Module Load
  • 영상 데이터에서 데이터 추출 및 정형화
    • 시작 프레임 : 20, 종료 프레임 : 59 설정 (약 1~3초 영상데이터 사용)




DataSet

  • 주어진 영상데이터의 길이가 모두 달라 모델을 구축하는데 어려움이 있어, 영상의 1~3초를 데이터로 추출
  • 대략 20프레임으로 구성, 총 40개 프레임
    • Landmark_ID : brazepose의 index를 나타냄
    • 3-dimensional axis
      • X-axis : 좌/우 움직임
      • Y-axis : 상/하 움직임
      • Z-axis : 원/근 움직임(카메라 기준)
    • Frame : 사진번호, 여기에서는 시간(time) 역할을 함
  • 문제점
    • 영상 데이터에서 각 데이터를 추출함에 있어 ‘영상에서 모델의 인식이 안되는 경우’, ‘뒤늦게 인식되는 경우’, ‘중간중간 끊김이 발생하는 경우’가 생김
    • 해당되는 데이터는 전부 제외시킴. 즉, 1~3초 온전한 프레임이 있는 데이터만 사용
모델 인식이 안되는 경우 뒤늦게 인식되는 경우 끈힉밍 발생하는 경우
  • 전체 프레임 이상 유/무 확인
  • 온전한 A017 데이터프레임 생성
  • 같은 방법으로, 온전한 A053 데이터프레임 생성

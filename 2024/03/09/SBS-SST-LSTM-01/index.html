SBS_SST_LSTM_01

|Post Views:

참고

  • (Step) 위성자료를 활용한 표층수온 예측 - Data Configuration

One Step : Lead 3 time

  • LSTM 모델을 활용하여 과거 데이터를 얼만큼 사용했을 때의 결과를 비교하기 위해 코드를 진행하기 앞서, 한 가지를 먼저 진행
  • 아래 내용에서는 Sequence Length = 3, 즉 3일간의 데이터를 사용했을 때의 결과를 먼저 비교하고자 함
    • 이후 Sequence Length의 변화에 따른 결과를 비교

Application with LSTM

  • Module Load
  • LSTM Model를 위한 Module Load
  • DataFrame Load
    • LSTM 모델을 돌리기 위해 생성하였던 데이터프레임 Load
  • LSTM 구현을 위해 Value Frame 생성
    • 시간을 나타내는 date, time column 삭제
  • shape 확인
    • date, time column 삭제 전 후 데이터프레임 확인
    • 삭제 전(main_df) : (366, 32002)
    • 삭제 후(SST_df) : (366, 32000)
  • LSTM을 구현하기 위해, 현재 데이터프레임에 생성되어있는 NaN column 제거
    • 모델을 돌리기 위한 데이터프레임에 NaN값이 포함되어있을 경우 동작하지 않는 것으로 알고 있다.
    • 현재 데이터프레임은 masking되어 있는 값으로 구분되기 때문에, masking patern을 가지고 있는 경우 제거하기 위해 column list를 작성하여 추가해준다.
  • Masking 된 Column을 제거하여 새로운 데이터프레임 생성
  • 최대, 최소 표층수온(Sea Surface Temperature) 설정
    • 차후 시각화하여 비교하기 위해 colorbar 설정 수치로 미리 변수 저장
  • LSTM 모델을 돌리기 위해, Sequence로 변환하는 함수 생성
  • Sequence Length를 지정하고, Sequence 생성
    • 여기서 Sequence Length란, 이전의 데이터를 얼마만큼 사용할 것인지를 의미
    • 따라서, 과거 몇 일의 데이터를 사용하여 예측할 것인지를 나타냄
  • Train/Test Set 분리
    • Train : Test = 8 : 2로 분리
  • 분리된 Dataset Shape 확인
  • LSTM 모델 생성 및 훈련
  • 구축된 모델을 기반으로 예측
  • 예측 값과 실제 값 간의 RMSE 구하기
    • 여기서 실제값은 OSTIA 관측 데이터를 의미

평균 제곱근 오차(Root Mean Square Error; RMSE)

  • 추정 값 또는 모델이 예측한 값과 실제 환경에서 관찰되는 값의 차이를 다룰 때 사용하는 측도
  • 관측 값과 예측 값 사이 오차의 절댓값을 활용한 지표

where $n$ is # of data, $y_i$ is reference of OSTIA, $x_i$ is prediction value

데이터 가공

예측 값과 실제 값 간의 비교를 위해 시각화를 진행

  • 예측 값과 실제 값의 각 데이터프레임 생성
  • 실제 값과 예측 값의 차이를 계산
  • 실제 값, 예측 값의 예측 날짜를 파악하기 위해, column을 매칭
    • 여기서 데이터는 일(Day) 단위로 따른다.
  • 데이터가 있는 값의 위도(Latitude), 경도(Longitude)를 추출
    • 이 코드의 경우, 각 데이터가 위, 경도에 대한 데이터가 최소 한 개씩 존재해서 Grid가 만들어지는 것일수도 있다.
    • 추후 한 번 더 고려해봐야 할 듯 하다.
  • Masking되어있는 부분을 추가하기 위해 기존 데이터프레임(SST_df)을 기준으로 column을 생성
    • SST_df : 메인 데이터프레임(main_df)에서 date(날짜), time(시간)을 제외한, value만 있는 데이터프레임
  • 실제값(y)에 NaN값 추가
  • reshape를 활용하여 데이터의 형태 변경
    • 현재 데이터의 경우 Row(시간)를 기준으로 Column별로 각 좌표의 표층수온 데이터가 저장되어 있다.
    • 이를 시각화하기 위해 (시간, lon, lat) 의 데이터의 형태로 변경할 필요가 있다.
    • 가장 중요한 작업. 그림 잘못나올 수도 있으니 주의
  • 만들어진 데이터의 형태 확인


실제 값(y) 시각화

  • 만들어진 데이터프레임을 바탕으로 시각화 진행
  • 아래 코드의 경우, 실제값(y)의 첫번째 day(23.10.20)을 시각화하여 표현
  • EDA 때 확인하였던 기존의 데이터와 그림이 일치
  • 위도, 경도, 만들어진 데이터의 형태가 일치하는지 확인


예측 값(prediction) 시각화

  • 위(실제 값 y)와 같은 방법으로 시각화 진행
  • Model 예측 결과를 시각화
기존 데이터 실제값(y) 예측값(prediction)


공간별 예측결과 차이

공간별 RMSE 시각화

  • RMSE 구하기
  • Masking된 좌표 추가 후, RMSE를 각 좌표별로 시각화

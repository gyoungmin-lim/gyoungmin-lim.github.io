(ML) IRIS Classification

Created2024-03-02|Updated2024-03-14|StudyML/DL
|Post Views:

머신러닝이란?

  • 명시적으로 프로그래밍하지 않고 컴퓨터에 학습할 수 있는 능력을 부여하는 학문
  • 경험에 쌓여 감에 따라 주어진 태스크의 성능이 점점 좋아질 때 컴퓨터 프로그램은 경험으로부터 학습한다고 할 수 있음


Process

지도학습

  • 학습 대상이 되는 데이터에 정답(label)을 붙여서 학습
  • 모델을 얻어서 완전히 새로운 데이터에 모델을 사용해서 ‘답’을 얻고자 하는 것




Iris Classification

IRIS

  • 프랑스의 국화
  • 아이리스(Iris)라는 이름은 그리스 신화의 무지개의 여신인 Iris에서 따온 것
    • 헤라 여신이 Iris에게 내린 축복의 숨결이 땅으로 떨어져 핀 꽃
    • 꽃말 : 좋은소식, 잘 전해주세요, 사랑의메시지


Goal

  • 아이리스의 품종 분류

    꽃잎(petal), 꽃받침(sepal)의 길이/너비 정보를 이용해서 이 3종의 품종을 구분할 수 있을까?


Checking Dataset

  • Load Iris Dataset
  • sklearn의 Dataset은 Python의 dict()형과 유사
  • iris key 확인
  • Iris Description 확인
  • target name 확인
  • target 확인
  • data 확인

  • 사람이 직접 구분하려면??
  • 일단, 먼저 versicolor, virginica, setosa를 구분하기 위해 3개 품종의 특성을 먼저 공부해야~

  • sepal width의 관계 확인

Pairplot

  • 꽃잎의 가로, 세로 길이를 가지고 더 높은 확률로 품종을 구분할 수 있을 것 같다.
  • 유의미할 것 같은 부분을 디테일하게 살펴보자.
    • 세 가지 품종에 대한 구분이 가능할 것 같음

Q. 품종 세 개를 구분할 수 있을까?

  1. Petal Length가 2.5보다 작다면 모두 Setosa로 분류해도 됨
  2. 몇 개의 데이터는 놓쳐지지만, petal width가 2.5보다 크면서 petal width가 1.6보다 작을 경우 Versicolor
  3. petal length가 2.5보다 크면서 petal width가 1.6보다 클 경우 Virginica
  • 현 상황에서 가장 논리적인 근거
  • 진행했을 때 각각의 차이점에 대한 정량적 수치 제시
  • 가로선이 어디에 있는 것이 최선인지 확인


Decision Tree

Decision Tree란?

  • Sklearn을 이용한 decision tree 구현
  • accuracy 확인


Back to Previous Story

  • Iris 품종을 분류하는 결정나무 모델이 어떻게 데이터를 분류했는지 알아보자.
  • Treemap 확인


과적합 (Overfitting)

  • Accuracy가 높게 나왔다고 해도 들여다 볼 필요가 있다.
  • 저 경계면은 올바른 걸까
    • 저 결과는 내가 가진 데이터를 벗어나서 일반화할 수 있는 걸까?
  • 어차피 얻은(혹은 구한) 데이터는 유한하고 내가 얻은 데이터를 이용해서 일반화를 추구하게 된다.
  • 이때 복잡한 경계면은 모델의 성능을 결국 나쁘게 만든다.




데이터 분리

  • 전체 데이터 (Origin Data) 중 모델 학습에 사용하지 않고 제외해 둔 데이터를 가지고 모델을 테스트한다.




(New) IRIS Classification

  • Train/Test 데이터를 8:2 비율로 분리
  • Train/Test 데이터가 잘 분리되었는지 확인
    • 문제가 각 클래스(setosa, versicolor, verginica) 별로 동일 비율이 아니다
  • 동일하게 분리하기 위해 stratify option 추가
  • Train data를 대상으로 decision tree 만들기
    • 학습할 때 마다 일관성을 위해 random_state만 고정
    • 모델을 단순화시키기 위해 max_depth를 조정
  • Treemap 확인
  • Train data에 대한 accuracy 확인
  • Train data에 대한 결정 경계 확인
  • Test data에 대한 Accuracy

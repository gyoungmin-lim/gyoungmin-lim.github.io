(Step) 위성자료를 활용한 표층수온 예측 - Buoy Data

Created2024-03-01|Updated2024-03-19|StudyStep By Step
|Post Views:

Dataset

  • 홈페이지 : 실시간 해양환경 어장정보시스템
  • 정선해양관측자료 다운로드 가능
    • 참고 : 한국 해양자료센터(Korea Oceanographic Data Center, KODC)


관측소 위치

  • 링크 : 관측소 검색
  • 운영여부 및 해역에 따라 설정 가능
  • 텍스트, 엑셀 지원


과거 관측정보

  • 과거 관측정보 다운로드


Select Buoy

  • Module Load
  • 다운로드받은 Buoy 위치 중, 불필요 자료 제거
  • Target Buoy 선정
    • 선정지역 내 존재하는 Buoy만 가져오기 위해 Location Boundary 설정
    • 경도 : 126.972 ~ 128.825
    • 위도 : 33.025 ~ 34.475
  • Boundary 내 Buoy만 가져오기

Target Buoy Data 저장


Buoy Data up to 2022

  • 해안 데이터의 경우 월별로 분리되어 있어, 이를 하나로 만들기 위해 파일 리스트를 호출하기 위한 변수 생성
  • 전체 수온 데이터 생성
  • 표층수온 column명 변경 및 불필요 column 제거

파일 저장


SST Data with Target Buoy

  • 위에서 선정한 Target Buoy를 바탕으로, 해당 기간 내 관측자료를 하나의 DataFrame으로 만듦
    • 기간 : 2021. 05. 51 ~ 2022. 12. 31
    • Location 내 Buoy 중, 종료 기간이 존재하지 않으며, 위 기간 내 운영하는 Buoy 8곳을 선정
  • 한글명으로 인한 오류를 방지하기 위해, column명을 한 번 더 변경
  • column 순서 변경
  • 관측소를 새로운 열로 변경하고, 각 관측소에 대한 표층수온 데이터를 해당 열에 매핑
    • 변경을 진행하며, time을 일(yy-mm-dd) 단위와 시간(hh:mm) 단위로 변경하여 각자 분리해줌

만들어진 데이터프레임 저장


Problem

  • 위에서 진행한 데이터 전처리의 경우, 2021-2022년 데이터만 존재
  • 2023년 데이터 전처리를 추가로 진행해야할 필요가 있음
  • 2023년 데이터의 경우, 실시간 관측정보를 통해서 얻을 수 있으며, 이는 파일 형태가 과거 관측정보와 다르다. (확장자 xlsx 및 각자 저장되어있어 별도의 전처리가 추가 필요)
  • 2023년 05월까지의 데이터프레임 생성


SST Data in 2023

  • Folder List 호출
    • 2023년 데이터의 경우, 각각의 관측소(Target Buoy)에 대한 실시간 관측정보로 받을 수 있어, 별도의 추가처리가 필요
    • Target Buoy = 고흥 동촌(fgdl4), 고흥 부아도(bgui5), 고흥 신촌(fgsl4), 고흥 연소(fgyl4), 완도 금일(wk094), 완도 동백(fwdf1), 완도 일정(fwih6), 장흥 회진(ejhfc)
  • Folder별로 Folder 내 ‘.xlsx’ 확장자 파일을 불러와 concat 진행
  • 과거 관측정보와의 통일성을 위해 column명 변경 및 필요정보만 추출
  • 과거 관측정보와 병합하기 위해 DataFrame 형태 변경

데이터 저장

  • 2023년 데이터로 새로 저장


전체 Buoy Data 생성

  • 2021-2022년 과거 관측정보와 2023년 실시간 정보를 하나의 DataFrame으로 병합
  • concat 활용 데이터프레임 병합

최종 SST data with Target Buoy DataFrame 생성

  • 다 사용한(불필요한) csv 데이터 파일 삭제 처리


Mean Imputation

  • 최종적으로 생성된 Buoy data의 경우 Missing data(NaN)가 존재함
    • 이번 Step에서는 Missing Data가 있을 경우, 그 Buoy를 제외한 나머지 Buoy data의 평균(Mean)으로 대체하는 것을 목표로 함
  • Figure Out을 위한 Module 추가 Load
  • Buoy Data Figure Storage 생성
  • 위에서 생성된 Buoy data Load
    • 아래 Table에서 보이는 것처럼 NaN값이 존재
  • 현재 time column의 경우 데이터 타입이 object로 구성되어 있다.
    • 시계열 그래프로 표현하기 위해 먼저 time column을 시간으로 변경
  • Buoy List 생성
    • 현재 Target Buoy로 설정한 리스트를 가져옴
    • ‘고흥 동촌(fgdl4)’, ‘고흥 부아도(bgui5)’, ‘고흥 신촌(fgsl4)’, ‘고흥 연소(fgyl4)’, ‘완도 금일(wk094)’, ‘완도 동백(fwdf1)’, ‘완도 일정(fwih6)’, ‘장흥 회진(ejhfc)’


Case 1. 특정 시간 기준 평균

  • 특정시간을 지정하여 해당 buoy가 NaN일 경우, 해당 buoy를 제외한 나머지 buoy값의 평균으로 Imputation 진행
  • 이 Case의 예시로는 특정시간을 09:00로 지정
  • Visualization
    • 위 Case 1 설정 진행 후, 결과를 확인하기 위해 시각화 진행
    • 비교대상 : 각 Buoy의 표층수온(SST)와 해당 Buoy를 제외한 표층수온(SST)의 평균
  • 코드가 작동하면서 해당 buoy를 제외한 buoy들의 평균값이 잘 계산되는지 확인
  • 평균 수온 비교 (예시)


Case 2. 일(Daily) 기준 평균

  • 일 평균을 구한 후, 해당 buoy가 NaN일 경우, 해당 buoy를 제외한 나머지 buoy값의 평균으로 Imputation 진행

  • 일 평균 구하기

  • 위와 마찬가지로, NaN값이 촌재할 경우 해당 Buoy를 제외한 나머지 Buoy의 평균값으로 대체
  • Visualization
    • 위 Case 2 설정에 따른 결과를 확인하기 위해 시각화 진행
    • 비교대상 : 각 Buoy의 표층수온(SST)과 해당 Buoy를 제외한 표층수온(SST)의 일 평균 비교
  • 평균 수온 비교 (예시)


Compare Cases

  • 위 두 가지 Case로 Imputation 진행한 결과를 비교
    • 특정시간을 Imputation한 결과와 일평균으로 변환 후 Imputation한 결과를 나타냄
    • 각 Buoy별로 구분하여 비교
      • 고흥 Buoy의 경우 전반적으로 유사한 그래프를 보임
      • 완도 Buoy의 경우 여름철에 차이가 나는 것을 확인
      • 장흥 Buoy의 경우 전반적으로 큰 차이를 보임
